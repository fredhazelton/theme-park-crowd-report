## 1. TikTok Landscape Research

## 2. Content Concepts (10 TikTok-Native Ideas)

### Concept 1: "Tomorrow's Crowds Predicted Today"
**Hook:** "Disney World will be a 9/10 busy tomorrow - here's why"
**Format:** 15-30 seconds, animated countdown + data visualization
**Data Source:** Next-day crowd predictions, weather data, events
**Frequency:** Daily (automated posting)
**Example Script:**
- 0-3s: Hook + tomorrow's crowd level (big text overlay)

- 10-15s: "When to visit instead" + CTA

**Hook:** "Watching these Disney ride wait times RACE in real-time"

**Frequency:** Weekly (different parks)

## 3. Technical Specifications

### Video Requirements
- **Dimensions:** 1080×1920px (9:16 aspect ratio) - MANDATORY
- **Duration:** 15-60 seconds optimal (can go up to 10 minutes, but short performs better)
- **File Size:** Max 2GB (aim for under 500MB for optimal upload)
- **Format:** MP4, MOV recommended
- **Frame Rate:** 24-30 fps
- **Audio:** Clear, trending sounds when possible

### Remotion Technical Specs
- **Resolution:** Export at 1080×1920 for best quality
- **Compression:** H.264 codec, high quality preset
- **Audio:** 44.1kHz, stereo
- **Text:** Large, readable fonts (minimum 24pt equivalent)
- **Animation:** Smooth 30fps animations, avoid quick cuts that might trigger seizure warnings

### Best Posting Times (EST)
**Primary Target (Theme Park Planners/Families):**
- **Tuesday-Thursday:** 6:00 PM - 10:00 PM (families browsing after dinner)
- **Sunday:** 7:00 PM - 9:00 PM (weekend planning)
- **Saturday:** 9:00 AM - 11:00 AM (vacation research time)

**Secondary (Disney Adults/Enthusiasts):**
- **Weekdays:** 12:00 PM - 1:00 PM (lunch break browsing)
- **Monday:** 8:00 PM - 10:00 PM (Monday blues + vacation dreaming)

## 4. Posting Cadence Options

### Option A: Conservative (3-4 posts/week)
**Schedule:**
- Monday: Crowd prediction for the week
- Wednesday: Educational/behind-the-scenes content
- Friday: Weekend crowd forecast
- Sunday: Weekly recap/data highlights (optional)

**Pros:**
- Manageable content creation load
- Quality over quantity focus
- Lower risk of audience fatigue
- Easier to maintain consistency

**Cons:**
- Slower audience growth
- Less algorithm exposure
- May miss trending opportunities
- Lower engagement volume

**Resource Requirements:** 2-3 hours/week content creation + automation setup

### Option B: Moderate (5-7 posts/week)
**Schedule:**
- Daily crowd predictions (Mon-Fri)
- 2x weekly educational/entertainment content (Tue, Thu)
- Weekend special content (Sat OR Sun)

**Pros:**
- Strong algorithm presence
- Good balance of utility + entertainment
- Regular audience touchpoints
- Sufficient content for growth

**Cons:**
- Higher production demands
- More automation complexity
- Need content backup plans

**Resource Requirements:** 4-5 hours/week + robust automation pipeline

### Option C: Aggressive (10-14 posts/week)
**Schedule:**
- Daily content minimum
- 2-3 times daily during peak seasons
- Real-time trending content
- Mix of evergreen + timely

**Pros:**
- Maximum algorithm exposure
- Rapid audience growth potential
- Capture all trending moments
- Dominate the Disney data niche

**Cons:**
- High resource demands
- Risk of content fatigue
- Quality control challenges
- Significant automation required

**Resource Requirements:** 8-10 hours/week + advanced automation + potential burnout

**RECOMMENDATION:** Start with Option B (Moderate) for 3 months, then scale to Option C based on performance and resource availability.

## 5. TikTok API Requirements

### Developer Account Setup
1. **TikTok Developers Portal:** Create account at developers.tiktok.com
2. **App Registration:** Required before accessing any APIs
3. **Business Verification:** May be required for Content Posting API access

### Content Posting API Access
**Current Status (2024-2025):**
- API exists but requires approval process
- Not publicly available to all developers
- Must demonstrate legitimate use case
- Review process can take 2-4 weeks

**Requirements for Approval:**
- Established business/brand presence
- Clear content strategy and use case
- Compliance with TikTok's community guidelines
- Technical integration plan

**API Capabilities:**
- Direct video posting
- Caption and hashtag management
- Basic video metadata
- Scheduled posting (limited)

**Limitations:**
- No live streaming API
- Limited creative tools access
- Rate limiting (exact limits undisclosed)
- Content must still follow all TikTok policies

### Alternative Posting Methods

**If API Access Denied/Delayed:**

1. **Browser Automation (Interim Solution):**
   - Use Puppeteer/Playwright to automate TikTok posting
   - Higher risk of detection/blocking
   - Requires careful rate limiting
   - Not recommended for long-term

2. **Manual Upload Pipeline:**
   - Remotion generates videos to folder
   - Team member uploads manually
   - Use TikTok Creator Tools for scheduling
   - Most reliable but labor-intensive

3. **Third-Party Tools:**
   - Later, Buffer, Hootsuite (TikTok integration limited)
   - Usually requires manual approval step
   - Higher costs but more features

**RECOMMENDATION:** Apply for TikTok Content Posting API immediately while developing manual backup system.
