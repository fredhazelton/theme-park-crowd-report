🔍 Who QA's Wilma?

The orchestrator needs oversight too
👑 Fred — Strategic QA
Reviews Wilma's strategic decisions — idea prioritization, resource allocation, phase transitions. Active during Discover & Grow.
👽 Gazoo — System Audit
Periodically audits Wilma's delegation patterns. Right tasks to right agents? QA ratings calibrated? Escalation frequency healthy?
📊 Adaptive Sampling
Same QA system applies to Wilma. T3 agents randomly sample her delegation decisions. Low-cost — verify the routing, not redo the work.

🖥️ Tier 4 Hardware Spec

Recommended Mac Studio configuration for local LLM inference

Configuration Options

BudgetM4 Max128GB70B Q8 / 72B Q6~15-20 tok/s~$3,500
★ RecommendedM3 Ultra256GB235B MoE Q8 / 70B FP16~18-20 tok/s~$7,000
MaximumM3 Ultra512GB405B Q4 / 671B MoE Q4~16-19 tok/s~$9,500
★ Recommended Model: Qwen3-235B-A22B (8-bit MLX)

Why Qwen3-235B: Mixture-of-Experts architecture — only 22B params active per token, so it runs fast despite 235B total. Benchmarks show PhD-level reasoning at ~19 tok/s on M3 Ultra 512GB. Best quality/speed balance for code + data science. Fits in 256GB at 8-bit (~234GB), leaving headroom for OS.

Alternatives worth benchmarking: DeepSeek V3 0324 (671B MoE, needs 512GB) · Llama 4 Maverick (fast on M3 Ultra) · Qwen3.5-35B-A3B (tiny active params, lightning fast, surprisingly capable)

For coding: Qwen3 Coder or DeepSeek Coder V2. Benchmark against real hazeydata tasks before committing.

Confidence tagging: Most modern LLMs can provide self-assessed confidence when prompted. Quality varies — typically useful as a triage signal (review low-confidence outputs first) even if imperfectly calibrated. Qwen3 and DeepSeek both support this via system prompts.
